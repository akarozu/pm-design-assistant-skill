# PMFrame 培训宣讲结构原则

## 核心教训：逻辑顺序决定说服力

**问题**：在给产品经理宣讲框架方法论时，内容罗列≠逻辑清晰。
常见错误：先展示结果（框架链、框架全景图），再讲方法论（为什么选这个框架）。

**正确顺序（Know Why → Know What → Know How）**：

```
第一章：Know Why（框架有什么用）
  → 建立动机：四大能力、五步工作流
  → 让学员理解"为什么要学这些框架"

第二章：框架全景（100个框架一览）
  → 建立全局认知：框架全景图（上/下）
  → 让学员知道"有哪些框架可用"

第三章：框架选择方法论【必须在框架全景之后，场景应用之前】
  → 核心缺失部分：三维决策模型（任务类型 × 产品类型 × 决策阶段）
  → 让学员掌握"怎么选对框架，而不是凭感觉"

第四章：高频场景框架链
  → 在全景和方法论之后，展示12个常见场景的框架链
  → 每行三段式：做什么（解决问题）→ 怎么做（操作步骤）→ 输出什么（交付物）

第五章：深度解析
  → 4个核心框架详解：服务蓝图、用户故事地图、RICE优先级、用户旅程地图
  → 每页三段式：做什么 → 怎么做 → 输出什么

第六章：实战案例
  → 完整案例走查，每步标注 [Step N · 框架名]

第七章：行动召唤（CTA）
```

## 为什么第三章（框架选择方法论）必须放在第三章

1. **先全景再方法论**：学员需要先知道"有哪些框架"（第二章），才能理解"怎么选"（第三章）。没有全景认知，选择无从谈起。
2. **先方法论再场景**：学员学会选择方法后，再用场景验证。场景框架链是方法论的应用，不是方法论本身。
3. **高频错误**：把"高频场景框架链"放在"框架全景图"和"选择方法论"之前——这是结果在前、原因在后，学员只能死记硬背，换个场景就不知道选什么框架。

## 框架详解页的标准结构

每个框架页（含案例页）统一使用三段式：

```
做什么（What）
  → 这个框架解决什么问题
  → 为什么在这个阶段用这个框架

怎么做（How）
  → 具体操作步骤（带编号）
  → 关键要点/常见陷阱

输出什么（Output）
  → 框架的交付物清单
  → 产出物的使用场景
```

## 案例页的特殊标注

每个案例步骤前加标签 `[Step N · 框架名]`，让学员清楚看到每一步用的是哪个框架：

```
[Step 0 · 读系统文档]
[Step 1 · 服务蓝图]
[Step 2 · 用户故事地图]
[Step 3 · RICE优先级]
[Step 4 · 用户旅程地图]
```

## 主题色：墨水经典

适用企业内宣、B端产品经理培训：
- 深色背景（墨色/深蓝黑）
- 纸张色文字（米白/淡黄）
- 强调色：淡金/琥珀色
- 衬线标题 + 无衬线正文

## 页数参考

完整版：20-22页
精简版：12-15页（覆盖核心逻辑线即可）
