## 一、银行RM工作台典型架构

### 1.1 双端结构

| 端 | 类型 | 用户 | 核心功能 |
|----|------|------|---------|
| **前端**（H5/小程序） | 客户经理工作台 | 客户经理 | 移动展业、数据查看、商机处理 |
| **后端**（B端SaaS） | 管理台 | 总行/分行/支行管理员 | 商户管理、政策配置、业绩统计 |

### 1.2 银行典型H5工作台结构

```
底部TabBar
├── 收单Tab（商户工作台）
├── 零售Tab（客户工作台）← 新增模块
├── 动态Tab（政策/培训/通知）
└── 我的Tab（个人信息）
```

## 二、模块扩展方法论

### 2.1 服务蓝图→断点识别

银行B端系统扩展时，核心断点类型：

| 断点类型 | 描述 | 典型处理 |
|---------|------|---------|
| **D1：框架先行** | 新Tab基础框架已就绪，数据源未接入 | MVP阶段先出UI结构，数据mock |
| **D2：数据源归属** | 同一数据需明确从哪个系统取数 | 需求阶段确认接口归属 |
| **D3：算力归属** | 复杂计算（绩效/佣金）的算力在哪里 | 明确系统边界 |
| **D4：消息统一** | 业务消息（商机/通知）是否合并展示 | 按业务分类独立/Tab过滤 |

### 2.2 数据源映射模式

银行系统典型数据归属：

| 数据类型 | 典型来源 | 备注 |
|---------|---------|------|
| 客户基本信息 | 大数据平台/CRM | 姓名/手机/AUM/产品持有 |
| 交易流水 | 核心系统 | 存款/贷款发生额 |
| 绩效数据 | 绩效系统 | 独立系统，需API对接 |
| 商机 | CRM | 业务线不同则分开推送 |
| 政策/培训 | 管理台动态管理模块 | 复用现有 |

### 2.3 RICE评分参考（银行B端场景）

| 因子 | 评分说明 |
|------|---------|
| **Reach** | 每季度受影响客户经理数（绝对数量） |
| **Impact** | 对核心KPI（AUM/绩效/客户增长）影响程度：0.25最小→3极高 |
| **Confidence** | 数据支撑100%/推断80%/猜测50% |
| **Effort** | 设计+工程+测试人月数 |

## 三、银行零售业务知识

### 3.1 绩效类型（参考案例）

| 绩效类型 | 主要业务 | 计算特点 |
|---------|---------|---------|
| **负债类** | 代销基金/保险/理财 | 按销售额×计提比例 |
| **资产类** | 贷款建档/投放 | 按户数×金额×系数 |
| **资产质量** | 重组/清收 | 按回收金额×计提比例 |
| **客户类** | 新增客户/产品配置 | 按户奖励，流失扣减 |

### 3.2 客户价值标签

| 标签 | AUM范围 | 说明 |
|------|---------|------|
| 基础户 | 1万≤AUM<10万 | — |
| 创富客户 | 10万≤AUM<50万 | — |
| 财富客户 | 50万≤AUM<300万 | — |
| 私行客户 | AUM≥300万 | — |

### 3.3 渠道签约类型

| 渠道 | 说明 |
|------|------|
| 手机银行 | 银行官方App |
| 微信动账通知 | 公众号推送 |
| 微信支付绑卡 | 微信生态内支付 |
| 云闪付绑卡 | 银联渠道 |
| 信用卡自动还款 | 借记卡自动扣款 |
| 生活缴费 | 水电煤等代缴 |

## 四、版本规划模式

### 4.1 MVP优先原则

银行B端功能版本划分：

| 版本 | 覆盖 | 判断标准 |
|------|------|---------|
| **MVP** | 核心闭环功能 | 日均触达60%+工作量的功能 |
| **V1.1** | 辅助管理功能 | 次高频但重要的功能 |
| **V2.0** | 增值功能 | 提升转化但非刚需的功能 |

### 4.2 典型扩展路径（参考案例）

```
现状：慧展业（收单模块）
↓ 扩展零售Tab
MVP：Tab框架 + 客户列表 + 客户详情 + 商机提醒
V1.1：绩效驾驶舱 + 业绩排行
V2.0：营销推送 + 完整画像
```

## 五、关键模式速查

### 5.1 商户→客户映射（收单→零售）

| 收单概念 | 零售对应 |
|---------|---------|
| 商户列表 | 客户列表 |
| 商户详情（经营数据） | 客户详情（资产总览） |
| 收单商户状态（未达标/已达标/已有效） | 客户标签（普通/重点） |
| 收单商机（管理台下发） | 零售商机（CRM推送，独立来源） |
| 补贴政策 | 绩效方案 |
| 动态消息（收单） | 动态消息（零售，暂分开） |

### 5.2 详情页组件模式

银行B端详情页典型布局：

```
┌─────────────────────────────┐
│  概览卡片（资产/KPI）         │
├─────────────────────────────┤
│  基本信息（来源大数据）        │
├─────────────────────────────┤
│  渠道签约（可推送）           │
├─────────────────────────────┤
│  产品配置（含到期提醒）        │
├─────────────────────────────┤
│  商机信息（关联CRM）          │
├─────────────────────────────┤
│  快速操作（拨号/标记/邀约）    │
└─────────────────────────────┘
```
