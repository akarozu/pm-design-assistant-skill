# 银行RM移动工作台 · 设计原则

> 来源：慧展业 RM移动展业工具需求分析实战（2026-05-20）
> 教训：首版方案以督导目标为核心 → 用户反馈「差距较远」→ 重新从RM日常工作出发

---

## 核心原则：效率工具 vs 考核工具

### 两种设计方向的根本区别

| 维度 | 管理视角（错误） | 客户经理视角（正确） |
|------|---------------|-------------------|
| **核心问题** | 「RM达标了吗？」 | 「RM今天干什么最有效？」 |
| **首页核心** | 渗透率完成率数字 | 今日待办 + 优先联系TOP5 |
| **数字展示** | 「定期渗透率 42%」 | 「还差8个客户买定期就达标了」 |
| **客户列表排序** | 按AUM排序 | 按「紧急程度+切入点」排序 |
| **成就感来源** | 排名数字 | 「本周成功挽留3个大额变动客户」 |
| **督导目标位置** | 首页大字看板 | 融入客户详情页，作为推荐依据而非监控面板 |

### 首页设计原则（已验证）

> **首页是「作战指挥台」而非「考核成绩单」**
> RM看到的是「今天打谁、怎么打」，而不是「我被扣了几分」

```
第一屏：
├── 今日概览（待联系N人 / 待兑现承诺X条 / 即将到期X人）
├── 优先联系 TOP5（客户名 + 优先级标签 + 切入话术1句）
└── 快捷入口（扫一扫/查客户/录跟进/活动工具）

第二屏：
├── 最新提醒（大额变动/产品到期/承诺到期）
├── 我的业绩摘要（渗透率差距 + 「还差X个达标」）
└── 本周进展（「本周已联系X人/转化X人」）
```

### 督导目标的正确嵌入方式

> 督导指标应该作为「数据底座」支撑RM决策，而非作为「首页看板」让RM被数字追着跑。

- **不放在首页顶部**（让人感觉被盯着）
- **融入客户详情页**（看到具体客户时，显示「这个客户买定期就能帮渗透率+0.3%」）
- **用「还差多少」而非「完成率」**（「差8个客户达标」比「42%完成率」更有行动感」）

## 问卷设计：RM视角调研方法论

### 设计原则

调研必须从「RM一天怎么干活」出发，不从「考核指标」出发。

**错误开场**：「您对渗透率指标的怎么看？」
**正确开场**：「您每天上班后第一个工作动作是什么？」

### 必问的五个维度

| 维度 | 核心问题 | 问卷题号示例 |
|------|---------|------------|
| 日常工作全流程 | RM一天怎么过的？ | Q1-Q10 |
| 信息与数据需求 | RM最想看到什么数据？ | Q11-Q13 |
| 商机优先级逻辑 | RM怎么决定今天联系谁？ | Q15-1~Q15-5 |
| 客户信息展示偏好 | 列表页/详情页应有哪些信息？ | Q15-6~Q15-10 |
| 场景化工具需求 | 5个核心场景痛点 | Q16-Q23 |

### 关键教训

- Q15-1~Q15-5（商机优先级）必须有：这是区分「系统推荐」vs「RM自主判断」的核心
- Q15-6~Q15-10（客户信息展示）必须有：列表页5项必选+详情页5项重点决定UI设计基础
- 督导指标调查要放在第二部分末尾，且用「你希望怎么展示」而非「你觉得重要吗」
