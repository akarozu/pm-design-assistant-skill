# #59 Hook上钩模型

## 核心概念

通过四步循环创造用户习惯：

```
触发（Trigger）→ 行动（Action）→ 多变奖励（Variable Reward）→ 投入（Investment）
     ↑                                                              ↓
     ←______________________用户开始期待下一次触发_______________________
```

## 四步详解

| 步骤 | 说明 | 案例 |
|------|------|------|
| **触发** | 外部触发（通知/邮件）或内部触发（无聊/孤独） | 微信红点、推送通知 |
| **行动** | 触发后用户完成的简单动作 | 打开App、滑动信息流 |
| **多变奖励** | 不确定的奖励让用户欲罢不能 | 点赞数不确定、刷不完的信息流 |
| **投入** | 用户投入时间/数据/社交资本，增加离开成本 | 发朋友圈、发红包、积累等级 |

## 使用时机

- C端消费品（社交/内容/娱乐）
- 需要建立用户习惯的产品
- 提高DAU/留存为目标

## ⚠️ 不适用条件（B端/低频/强制使用场景必须先读）

| 场景 | 风险 | 替代框架 |
|------|------|---------|
| **低频工具/报表产品** | 用户使用频率由业务周期决定，不是产品体验能改变的 | RICE + 用户故事地图 |
| **B端审批流** | 审批员不是"期待"触发而是"被迫"处理，行为动机完全不同 | 服务蓝图 + DACI |
| **强制使用场景** | 用户使用由监管/组织规定，不是"习惯"能解释的 | 用户旅程地图（任务完成视角） |
| **监管合规类产品** | 合规要求的使用行为不存在"期待"空间 | 直接用功能清单+RICE |
| **B端长周期决策产品** | 决策周期长，"习惯"框架不适用 | 服务蓝图 + RICE |

> **低频产品判断标准**：如果用户使用产品的频率由外部业务周期决定（日报/月报/季报），而非由产品体验决定，则判定为低频产品，Hook模型不适用。
>
> **强制使用判断标准**：如果用户使用产品是因为组织/监管强制要求，而非个人意愿，则行为动机分析框架（Hook/Fogg）不适用。
